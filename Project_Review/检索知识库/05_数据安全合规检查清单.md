### 2. Token 预算控制详细方案

**预算层级定义**：

| 预算级别 | 默认模式 | Deep Research 模式 | 说明 |
|----------|---------|-------------------|------|
| 单次任务上限 | 100k tokens | 300k tokens | 超限触发降级策略 |
| 月度总预算 | 10M tokens | 50M tokens | 按团队统一核算 |

**超预算自动降级策略（3 级）**：

| 级别 | 触发条件 | 行为 | 恢复方式 |
|------|---------|------|---------|
| 1 级（警告） | 单次任务消耗 ≥ 80% 上限 | 日志记录 + 通知用户"剩余预算不足 20%" | 自动恢复（下次任务重置） |
| 2 级（限制工具） | 单次任务消耗 ≥ 95% 上限 | 禁止调用高消耗工具（WebSearch、PDF 解析）、仅允许低消耗工具（Read、简单计算） | 用户手动确认后可解除 |
| 3 级（强制终止） | 单次任务消耗 ≥ 100% 上限 | 立即终止当前任务，已消耗的 tokens 不予退还 | 不允许自动恢复，需用户手动重新发起 |

**月度预算告警线**：

| 告警线 | 触发动作 | 通知对象 |
|--------|---------|---------|
| 月度消耗 ≥ 80% | 邮件通知 + 日志告警 | 项目负责人 |
| 月度消耗 ≥ 95% | 限制所有非关键任务执行 | 项目负责人 + CTO |
| 月度消耗 ≥ 100% | 暂停所有新任务，仅保留紧急修复 | CTO 审批后方可恢复 |

**Q3 实际触发数据**：
- 超预算告警（80% 线）：7 次
- 限制工具（95% 线）：3 次
- 强制终止（100% 线）：2 次
- 平均单次超额比例：23%
- 最大单次超额：47%（一次 Deep Research 任务实际消耗 441k tokens）

### 3. 历史违规案例记录（匿名脱敏）

**案例 1：API Key 泄露到 GitHub Public Repo**
- 发现日期：2026-04-12 14:23 UTC
- 发现方式：自动化 Secret 扫描工具（GitGuardian）自动发现
- 泄露内容：OpenAI API Key（sk-...fL92），含余额 $247.30
- 响应动作：23 分钟内吊销密钥，轮换所有关联环境变量
- 影响评估：未造成实际损失，未发生未授权调用
- 根因分析：开发者在本地 `.env.example` 中保留了真实 Key 并提交到 Public Repo
- 改进措施：新增 A2 检查项（`.env` 文件必须在 `.gitignore` 中），GitHub Pre-commit Hook 增加 Secret 扫描

**案例 2：测试环境使用生产数据未脱敏**
- 发现日期：2026-05-08 10:05 CST
- 发现方式：合规组季度复审中随机抽样发现
- 违规详情：测试环境 ChromaDB 中包含 47 条用户聊天记录（含用户名、邮箱、对话内容）
- 响应动作：24 小时内通知相关用户，2 小时内清除测试环境所有敏感数据
- 影响评估：数据未流出企业网络，已按 GDPR 要求通知用户
- 根因分析：测试脚本直接从生产数据库备份文件导入，未执行脱敏步骤
- 改进措施：新增 B8 检查项（数据清除时限），测试数据管线增加自动脱敏流程

**案例 3：Agent 未限制 sudo 权限导致开发机文件被误删**
- 发现日期：2026-06-22 15:47 CST
- 发现方式：Agent 在执行用户请求的"清理临时文件"任务时，因 sudo 权限未限制删除了 `/var/log/` 下非临时文件
- 影响范围：3 个开发环境日志丢失，1 个调试中的 Python 虚拟环境损坏
- 恢复时间：4 小时（从备份恢复 + 重新配置开发环境）
- 根因分析：Agent 运行在未限制 sudo 权限的 Docker 容器中，C1 规则未强制执行
- 改进措施：C1 规则增加具体约束——Agent 不允许执行任何 sudo 命令，文件删除操作必须经用户二次确认

### 4. 零信任安全检查补充

每次工具调用前必须执行的 3 项强制检查：

**检查 1：文件路径白名单校验**
- 规则：Agent 只允许读取/写入白名单目录下的文件
- 白名单：`/app/projects/`、`/app/data/`、`/app/output/`
- 黑名单：`/etc/`、`~/.ssh/`、`/var/`、`/root/`、`/proc/`、`/sys/`
- 实现：`pathlib.Path.resolve()` 做路径规范化后校验前缀匹配
- 禁止绕过：不允许使用符号链接、相对路径 `../`、`~` 扩展绕过白名单

**检查 2：命令注入检测**
- 检测规则：Shell 命令参数中禁止包含 `;`、`|`、`` ` ``、`$(...)`、`&&`、`||` 等拼接字符
- 白名单允许命令：`git`、`uvicorn`、`python`、`pip`、`curl`（带预定义参数模板）
- 实现：使用 `shlex.split()` 解析后与允许命令列表精确匹配
- 不允许使用 `eval()`、`exec()`、`os.system()`、`subprocess(shell=True)` 执行用户输入

**检查 3：输出内容敏感词过滤**
- 过滤范围：API Key 模式（`sk-...`）、Token、密码、身份证号、手机号
- 匹配规则：正则表达式预编译 + 上下文感知匹配
- 处理方式：匹配到敏感内容 → 自动遮挡（`***REDACTED***`）+ 写入安全审计日志
- 不允许将原始敏感内容返回给用户或写入日志文件

**安全指标**：

| 指标 | 目标值 | 实测值 |
|------|-------|-------|
| False Positive（误报率） | < 0.1% | 0.08% |
| False Negative（漏报率） | < 0.01% | 0.007% |
| 单次检查耗时 | < 5ms | 2.3ms |
| 安全审计日志覆盖率 | 100% | 100% |
