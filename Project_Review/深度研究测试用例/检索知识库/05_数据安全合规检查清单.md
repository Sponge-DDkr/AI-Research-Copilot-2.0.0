# 数据安全合规检查清单

> 版本：v2.1 | 生效日期：2026-07-01 | 合规负责人：周安全
> 适用对象：所有 AI 项目上线前必检

---

## 检查说明

- 每条检查项目三个状态：✅ 已通过 / ❌ 未通过 / ⬜ 不适用
- ❌ 项目必须在 **上线前** 全部修复，不得有例外
- 项目负责人签字确认，合规组存档

---

## A. API 与密钥安全（共 8 项）

| # | 检查项 | 严重级别 |
|---|--------|---------|
| A1 | API Key 不得硬编码在源代码中 | 🔴 严重 |
| A2 | `.env` 文件必须在 `.gitignore` 中 | 🔴 严重 |
| A3 | 生产环境 API Key 必须使用 K8s Secret 或云 Secret Manager | 🔴 严重 |
| A4 | API Key 不得出现在日志/错误堆栈中 | 🟠 高 |
| A5 | 所有外部 API 调用必须启用 HTTPS | 🔴 严重 |
| A6 | LLM API 调用必须记录审计日志（脱敏后） | 🟠 高 |
| A7 | API 端点必须有限流（Rate Limiting） | 🟡 中 |
| A8 | 管理类 API（/admin/*）必须有额外认证层 | 🔴 严重 |

---

## B. 用户数据保护（共 10 项）

| # | 检查项 | 严重级别 |
|---|--------|---------|
| B1 | 用户输入内容不得用于模型训练（除非用户明确同意） | 🔴 严重 |
| B2 | PII（手机号、身份证、邮箱）必须自动脱敏展示 | 🔴 严重 |
| B3 | 用户密码使用 bcrypt（cost≥12）或 argon2 哈希存储 | 🔴 严重 |
| B4 | 用户可自助导出全部个人数据（GDPR 合规） | 🟠 高 |
| B5 | 用户可自助删除账号及关联数据（48h 内执行） | 🟠 高 |
| B6 | 用户对话历史默认保留 30 天，超期自动清除 | 🟡 中 |
| B7 | Cookie / Token 设置 HttpOnly + Secure + SameSite=Strict | 🟠 高 |
| B8 | 用户上传的文档在删除后 7 天内从 ChromaDB 和文件系统彻底清除 | 🟠 高 |
| B9 | 不允许收集 13 岁以下儿童的个人信息 | 🔴 严重 |
| B10 | 跨用户数据隔离（用户 A 不能检索到用户 B 的文档） | 🔴 严重 |

---

## C. Agent 安全约束（共 6 项）

| # | 检查项 | 严重级别 |
|---|--------|---------|
| C1 | Agent 工具调用必须有黑名单（禁止：文件删除、Shell、系统配置修改） | 🔴 严重 |
| C2 | Agent 生成的代码/命令在执行前必须经用户确认 | 🔴 严重 |
| C3 | Agent 调用外部 URL 必须检查域名白名单 | 🟠 高 |
| C4 | Agent 最大迭代次数必须有硬限制（防止死循环耗尽预算） | 🟠 高 |
| C5 | Agent 单次任务 token 消耗上限（防止 prompt injection 攻击耗尽额度） | 🟡 中 |
| C6 | 用户可通过「紧急停止」按钮中断正在运行的 Agent | 🟡 中 |

---

## D. 模型与输出安全（共 5 项）

| # | 检查项 | 严重级别 |
|---|--------|---------|
| D1 | LLM 输出必须经过内容安全过滤（色情/暴力/违法/政治敏感） | 🔴 严重 |
| D2 | RAG 检索结果必须做来源标注（区分「来自文档」vs「LLM 知识」） | 🟠 高 |
| D3 | 不得在 System Prompt 中硬编码任何公司机密信息 | 🟠 高 |
| D4 | 对外输出的报告必须标注"由 AI 生成，仅供参考" | 🟡 中 |
| D5 | 模型输出的可执行代码必须经过沙箱验证 | 🟡 中 |

---

## E. 基础设施安全（共 4 项）

| # | 检查项 | 严重级别 |
|---|--------|---------|
| E1 | 数据库（SQLite/PostgreSQL）文件访问权限限制为应用用户 only | 🟠 高 |
| E2 | ChromaDB 持久化目录权限限制，不允许外部读取 | 🟠 高 |
| E3 | Docker 容器不允许以 root 权限运行 | 🟠 高 |
| E4 | 启用 HTTPS（生产环境必须，开发环境可选） | 🔴 严重 |

---

## F. 合规与审计（共 3 项）

| # | 检查项 | 严重级别 |
|---|--------|---------|
| F1 | 所有检查项结果归档保存，保留至少 2 年 | 🟡 中 |
| F2 | 每季度进行一次合规复审 | 🟡 中 |
| F3 | 发现安全事故后 24 小时内上报合规组 + CTO | 🔴 严重 |
