## 二、Q3 完成的重点项目

### 项目 A：智能报告生成引擎升级（代号「笔杆子」）

- 上线时间：2026-08-15
- 负责人：王前端 + 钱后端
- 核心改动：从固定模板升级为 LLM 动态生成，支持 12 种报告类型
- 用户反馈评分：4.7/5.0
- **关键决策**：放弃了自研模板引擎，改用 LLM + JSON Schema 约束输出格式。理由是自研模板维护成本太高（历史上 12 种报告类型 × 3 个版本的排列组合导致 36 套模板的维护噩梦）。
- 技术细节：使用了 DeepSeek V3 的 JSON Mode，配合 Pydantic 做输出校验，不合规的输出自动重试（最多 2 次）。

### 项目 B：Agent Timeline 可视化重构

- 上线时间：2026-09-05
- 负责人：赵前端
- 核心改动：用 SSE 事件驱动的时间线组件替代旧的静态进度条
- 用户满意度从 3.2 提升到 4.6
- 技术栈：React 19 + TailwindCSS 4 + `react-markdown`
- **最大的坑**：SSE 断连重试。最初用 `EventSource` API，但它不支持 POST + 自定义 header。后来改为 `fetch` + `ReadableStream` + 手动行解析，断连后 3 秒自动重连。

### 项目 C：企业知识库权限系统

- 上线时间：2026-09-28
- 负责人：孙后端
- 支持按部门/角色/个人三级权限控制
- 技术方案：每个文档 chunk 的 metadata 里带 `access_level` 标签（`public` / `team` / `dept` / `private`），检索时按用户权限做 metadata filter
- **安全审计**：通过外部安全公司渗透测试，0 高危漏洞

## 三、未达标的项目

### 项目 D：PDF 复杂表格解析

- 原计划 Q3 完成，延迟到 Q4
- 根因：非结构化 PDF（扫描件、合并单元格）的表格解析准确率只有 72%，不满足 95% 的基线要求
- 尝试了 3 种方案：PyMuPDF → Camelot → Unstructured.io → 最终选定 Azure Document Intelligence（准确率 96%，但成本 ¥0.15/页）
- **教训**：不要高估开源工具对中文复杂排版的解析能力。预算允许的情况下，付费 API 省的时间远超成本。
